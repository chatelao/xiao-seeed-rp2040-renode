# RP2350 Gap Analysis Report

This document analyzes the technical gaps between the Raspberry Pi RP2040 and the RP2350 in the context of Renode simulation support.

## 1. Architectural Overview

| Feature | RP2040 | RP2350 |
| --- | --- | --- |
| **CPU Cores** | 2x Cortex-M0+ (133 MHz) | 2x Cortex-M33 (150 MHz) + 2x Hazard3 (RISC-V) |
| **SRAM** | 264 KB | 520 KB |
| **OTP** | n/a | 8 KB |
| **Security** | n/a | TrustZone, SHA256, TRNG, Signed Boot |
| **GPIOs** | 30 | Up to 48 (RP2350B) |

## 2. Key Peripheral Gaps

### 2.1 CPU & SIO
- **Cortex-M33 & Hazard3:** RP2350 introduces dual Cortex-M33 (Armv8-M) and dual Hazard3 (RISC-V) cores. Renode supports Cortex-M33 generally, but the Hazard3 core and its specific extensions are not publicly available in Renode.
- **SIO Changes:**
    - **Hardware Divider:** Removed from SIO; replaced by CPU-native instructions.
    - **Doorbells:** Added 8 inter-core doorbells for synchronization.
    - **GPIO Co-processor:** Added for low-latency GPIO access.

### 2.2 Memory & Boot
- **Increased SRAM:** Increased to 520 KB.
- **OTP Memory:** 8 KB of One-Time Programmable memory added for security and configuration.
- **Bootrom:** New features for signed image verification and encrypted boot.

### 2.3 Programmable I/O (PIO)
- **PIO v1:** 3 PIO blocks (vs 2) with 4 state machines each (total 12 SMs).
- **Expanded Pins:** Support for addressing up to 48 GPIOs.

### 2.4 DMA & Interrupts
- **DMA Channels:** Increased to 16 channels (vs 12).
- **DMA IRQs:** Increased to 4 interrupt lines (vs 2).

### 2.5 PWM & Timer
- **PWM Slices:** Increased to 12 slices (vs 8).
- **PWM IRQs:** Increased to 2 interrupt lines (vs 1).
- **Second Timer:** Added a second Timer instance (`timer1`).

### 2.6 RTC & Power Management (POWMAN)
- **RTC Removal:** The RP2040-style RTC is removed.
- **POWMAN:** New Power Manager peripheral with an Always-On (AON) timer.

### 2.7 New Hardware Blocks
- **SHA256:** Dedicated hardware for SHA256 hashing.
- **TRNG:** True Random Number Generator.
- **HSTX:** High-Speed Serial Transmit peripheral for driving high-speed interfaces like DVI.

## 3. Renode Support Status

As of May 2026, public support for the RP2350 in Renode is limited:
- **Cortex-M33:** General support exists, but RP2350-specific implementation (e.g., TrustZone configuration, double co-processor) is missing.
- **Hazard3:** No public model for the Hazard3 RISC-V core was found.
- **Peripherals:** Models for RP2350-specific peripherals (POWMAN, SHA256, TRNG, HSTX) and the updated SIO/PIO/DMA/PWM registers are not yet available in the public Renode repositories.

## 4. Implementation Recommendations

To support RP2350 in the current simulation environment, the following steps are required:
1. **Core Integration:** Add support for the Hazard3 RISC-V core and configure the Cortex-M33 with RP2350-specific parameters.
2. **Memory Map Update:** Update the `.repl` files to reflect the 520 KB SRAM and the new peripheral base addresses.
3. **SIO Refactoring:** Update the `RP2040SIO` model to remove the hardware divider (for RP2350) and add the doorbell/GPIO co-processor logic.
4. **Peripheral Evolution:**
    - Update `RPDMA`, `RP2040PIO`, and `RP2040PWM` to support the increased channel/SM/slice counts and IRQ mappings.
    - Implement the `RP2350Powman` model to replace the `RP2040RTC`.
5. **New Peripherals:** Implement stubs or functional models for SHA256, TRNG, and HSTX as needed for firmware verification.
